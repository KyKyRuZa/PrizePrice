# План исправления мониторинга PrizePrice

## Решения (приняты)

1. **Сеть**: Подключить Prometheus и Grafana к сети `prizeprice`, target изменить на `server:3001`
2. **cAdvisor**: Добавить сервис cAdvisor в docker-compose.monitoring.yml
3. **Аутентификация**: Использовать Bearer token для /metrics endpoint
4. **Алерты**: Переместить alerts.prometheus.yml в infra/monitoring/
5. **Активные пользователи**: Реализовать метрику (Gauge) — определить логику подсчёта
6. **Дубликат**: Удалить server/src/utils/metrics.js

---

## Пошаговый план реализации

### Шаг 1: Исправить путь к alerts.prometheus.yml

**Файлы для изменения:**
- `docs/MONITORING.md` — обновить путь на `../monitoring/alerts.prometheus.yml`
- `server/README.md` — обновить путь на `../monitoring/alerts.prometheus.yml`

**Действия:**
```bash
mv docs/monitoring/alerts.prometheus.yml infra/monitoring/
```

### Шаг 2: Настроить аутентификацию в Prometheus

**Файл:** `infra/monitoring/prometheus.yml`

Изменить job `prizeprice-api`:
```yaml
- job_name: 'prizeprice-api'
  metrics_path: /metrics
  scheme: http
  bearer_token: ${METRICS_TOKEN}
  static_configs:
    - targets: ['server:3001']
```

### Шаг 3: Подключить мониторинг к сети приложения

**Файл:** `infra/monitoring/docker-compose.monitoring.yml`

Изменить networks:
```yaml
networks:
  monitoring:
    external: true
    name: prizeprice  # для prod
    # или prizeprice-dev для dev
```

Убрать `extra_hosts` (больше не нужен).

### Шаг 4: Добавить cAdvisor сервис

**Файл:** `infra/monitoring/docker-compose.monitoring.yml`

Добавить сервис:
```yaml
cadvisor:
  image: gcr.io/cadvisor/cadvisor:latest
  container_name: monitoring-cadvisor
  ports:
    - "8080:8080"
  volumes:
    - /:/rootfs:ro
    - /var/run:/var/run:ro
    - /sys:/sys:ro
    - /var/lib/docker/:/var/lib/docker:ro
  networks:
    - monitoring
  privileged: true
```

### Шаг 5: Реализовать active_users_total

**Файл:** `server/src/middlewares/metrics.middleware.js`

Определить метрику:
```javascript
export const activeUsersTotal = new client.Gauge({
  name: 'active_users_total',
  help: 'Number of currently active users',
});
```

**Логика обновления (Вариант B — фоновый таймер):**
- Запустить setInterval каждые 30 секунд
- Запрос в БД: `SELECT COUNT(*) FROM users WHERE last_seen > NOW() - INTERVAL '5 minutes'`
- Обновлять метрику: `activeUsersTotal.set(count)`

**Требует:**
- Подключение к БД в фоновой задаче (использовать существующий pg-пул из `server/src/utils/pg.js`)
- Импорт метрики в файл фоновой задачи (например, в `preflight/index.js` или отдельный файл `server/src/utils/activeUsersUpdater.js`)

**Примечание:**last_seen должен обновляться при каждом запросе авторизованного пользователя (уже есть в `requestContext.middleware.js`).

### Шаг 6: Удалить дубликат

```bash
rm server/src/utils/metrics.js
```

Проверить, нет ли импортов из этого файла.

### Шаг 7: Обновить env-файлы

Добавить `METRICS_TOKEN` в:
- `infra/env/.env.dev`
- `infra/env/.env.prod`

Значение: сгенерировать безопасный токен (например, через `openssl rand -hex 32`).

### Шаг 8: Обновить документацию

- `docs/MONITORING.md` — описать новую сетевую конфигурацию
- `server/README.md` — обновить пути к alerts

## Проверка

1. Запустить мониторинг: `cd infra/monitoring && docker compose -f docker-compose.monitoring.yml up -d`
2. Проверить Targets в Prometheus (http://localhost:9090/targets) — все должны быть UP
3. Проверить, что Grafana загружает дашборд без ошибок
4. Проверить алерты: `curl http://localhost:9090/api/v1/rules`
5. Убедиться, что метрики приходят: `curl http://localhost:3001/metrics` (с токеном)

## Принятые решения

1. **Сеть**: Подключить Prometheus и Grafana к сети `prizeprice` (production only). Target: `server:3001`
2. **cAdvisor**: Добавить сервис cAdvisor в docker-compose.monitoring.yml
3. **Аутентификация**: Использовать Bearer token (`METRICS_TOKEN`) для /metrics endpoint
4. **Алерты**: Переместить `alerts.prometheus.yml` из `docs/monitoring/` в `infra/monitoring/`
5. **Активные пользователи**: Реализовать Gauge-метрику `active_users_total` с фоновым обновлением каждые 30 секунд (запрос в БД по полю `last_seen`)
6. **Дубликат**: Удалить `server/src/utils/metrics.js` (неиспользуемый)
7. **Окружение**: Monitoring stack запускается только в production, dev-окружение не использует отдельный мониторинг

---

## Scope

- **Production**: Полный monitoring stack (Prometheus + Grafana + cAdvisor)
- **Development**: Мониторинг не запускается; разработчики могут использовать локальный Prometheus при необходимости
